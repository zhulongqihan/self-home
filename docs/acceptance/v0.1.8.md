# 验收清单 v0.1.8 - 订单状态机 + 流转 API

## 本次改动
- 后端：订单状态合法流转校验、`PATCH /api/orders/:id/status`、`PATCH /api/orders/:id/reply`
- 后端：`GET /api/orders/owner/all`（店长）、`GET /api/orders/:id`（详情）
- 店长端：订单管理页（列表、接单/推进状态、备注）
- 顾客端：订单 Tab 展开进度时间线；待接单可取消；展示店长备注

## 状态流转（店长）
待接单 → 已接单 → 制作中 → 配送中 → 已送达 → 待评价 → 已完成

店长可在「待接单 / 已接单」阶段取消订单。

## 如何体验
1. 微信开发者工具重新编译
2. 顾客 `zhuzhu` 在厨房下一单
3. 顾客「订单」Tab：点开订单看进度；待接单时点「取消订单」
4. 店长 `yangyang` → 工作台「订单管理」
5. 点开订单 → 「接单」→ 依次推进至「完成订单」
6. 填写店长备注并保存，顾客订单页应显示「店长说」

## 验收点
- [ ] 顾客下单后订单状态为「待接单」
- [ ] 店长可接单并依次推进状态，非法跳转被拒绝（接口 400）

- [ ] 店长备注保存后顾客可见
- [ ] 订单详情/列表展示状态时间线
- [ ] blog / agent 站点仍正常
